# 回撤计算逻辑修复说明

## 📋 问题描述

用户反馈：在2025/9/20到2025/9/27期间，实际运行的回撤值超过了日志中显示的"回撤371元"预期值。

## 🔍 根本原因

发现两个问题：

### 1. 硬编码的预期值不准确
- **问题代码**（第3558行）：
  ```python
  self.log_output(f"暂停策略: ROI 18.46% | 回撤371元 | 净利润+1476元 ⭐推荐\n\n")
  ```
- **问题**：371元是历史测试数据，但未标注，导致用户误以为是固定值
- **影响**：用户期望回撤≤371元，但实际可能更高

### 2. 回撤计算逻辑不完整
- **问题代码**（第3635-3641行）：
  ```python
  if hit:
      # ... 命中处理
      self.fib_index = 0
  else:
      # ... 未中处理
      self.fib_index += 1
      
      if self.balance < self.min_balance:  # ❌ 只在else分支更新
          self.min_balance = self.balance
          self.max_drawdown = abs(self.min_balance)
  ```
- **问题**：回撤检查只在未命中分支执行，导致以下场景计算错误：
  - 连续亏损后小幅命中，余额仍为负数
  - 命中后余额虽有增加但仍低于之前的最低点

## ✅ 修复方案

### 修复1：标注历史参考值
**修改位置**：第3554-3559行

**修改前**：
```python
self.log_output(f"核心参数: 窗口12期 | 增强≥35%×1.5 | 降低≤20%×1.0 | 命中后暂停1期\n")
self.log_output(f"基础表现: ROI 12.70% | 回撤742元 | 净利润+1374元\n")
self.log_output(f"暂停策略: ROI 18.46% | 回撤371元 | 净利润+1476元 ⭐推荐\n\n")
```

**修改后**：
```python
self.log_output(f"核心参数: 窗口12期 | 增强≥35%×1.5 | 降低≤20%×1.0 | 命中后暂停1期\n")
self.log_output(f"历史测试参考（实际结果可能有所不同）：\n")
self.log_output(f"  基础表现: ROI 12.70% | 回撤742元 | 净利润+1374元\n")
self.log_output(f"  暂停策略: ROI 18.46% | 回撤可变 | 净利润+1476元 ⭐推荐\n")
self.log_output(f"注意：回撤值受数据周期影响，以实际运行结果为准\n\n")
```

**改进**：
- ✅ 明确标注"历史测试参考"
- ✅ 说明"实际结果可能有所不同"
- ✅ 将"回撤371元"改为"回撤可变"
- ✅ 添加注意事项

### 修复2：完善回撤计算逻辑
**修改位置**：第3628-3644行

**修改前**：
```python
if hit:
    win = self.cfg['win_reward'] * multiplier
    self.total_win += win
    profit = win - bet
    self.balance += profit
    self.fib_index = 0
else:
    profit = -bet
    self.balance += profit
    self.fib_index += 1
    
    if self.balance < self.min_balance:  # ❌ 只在else分支
        self.min_balance = self.balance
        self.max_drawdown = abs(self.min_balance)
```

**修改后**：
```python
if hit:
    win = self.cfg['win_reward'] * multiplier
    self.total_win += win
    profit = win - bet
    self.balance += profit
    self.fib_index = 0
else:
    profit = -bet
    self.balance += profit
    self.fib_index += 1

# 更新最大回撤（无论命中还是未命中都要检查）✅
if self.balance < self.min_balance:
    self.min_balance = self.balance
    self.max_drawdown = abs(self.min_balance)
```

**改进**：
- ✅ 将回撤检查移到if-else之外
- ✅ 无论命中还是未命中都检查回撤
- ✅ 确保捕获所有可能的最低余额点

## 📝 用户建议

1. **查看实际结果**
   - 不要依赖硬编码的"371元"预期值
   - 以实际运行后的"最大回撤"数值为准

2. **理解回撤含义**
   - 回撤 = 从最高点到最低点的跌幅
   - 连续未中期间回撤会快速增加
   - 即使命中后余额回升，回撤仍保持历史最大值

3. **风险控制**
   - 建议准备至少 2倍最大回撤 的资金
   - 根据实际数据周期评估风险
   - 不同时间段数据可能导致不同回撤值
